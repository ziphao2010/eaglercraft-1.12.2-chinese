# Eaglercraft 1.12.2 中文版 - Git 提交信息规范

## 提交格式
使用中文编写提交信息，遵循以下格式：
```
<类型>: <简短描述>

<详细描述>
```

## 类型说明
- `feat`: 新功能或新特性
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式调整（不影响功能）
- `refactor`: 重构代码
- `test`: 测试相关
- `chore`: 构建/工具链相关
- `i18n`: 国际化/翻译更新
- `lang`: 语言文件更新

## 要求
- 使用清晰简洁的中文
- 首字母大写，结尾不加标点
- 详细描述可选，但较大变更需说明
- 对于语言文件更新，说明更新了哪些内容

## 示例
```
feat: 添加中文语言支持
- 完善 zh_CN.lang 翻译
- 优化游戏内文本显示

fix: 修复启动脚本端口冲突检测
- 改进错误提示信息
- 添加更友好的用户反馈

i18n: 更新简体中文翻译
- 修正物品名称翻译
- 完善游戏菜单文本
```
